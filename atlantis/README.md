# Atlantis - Terraform Automation for Pull Requests

Atlantis automates Terraform workflows through GitHub Pull Requests, enabling collaborative infrastructure changes with built-in review and approval processes.

## Overview

This Atlantis deployment provides:

- **Automated Planning**: Runs `terraform plan` automatically on PRs
- **PR-Based Applies**: Apply infrastructure changes via PR comments
- **Multi-Workspace Support**: Manages multiple environments (sandbox, dev, prod)
- **Dynamic Configuration**: Auto-discovers Terraform projects in the repository
- **GitHub App Integration**: Secure authentication via GitHub App
- **AWS Integration**: IRSA for secure AWS access without credentials

## Quick Start

### Prerequisites

- EKS cluster running
- AWS Load Balancer Controller installed
- GitHub App created and installed
- AWS Secrets Manager secrets configured

### Deploy Atlantis

```bash
cd atlantis/terraform

# Initialize Terraform
terraform init

# Select or create workspace
terraform workspace select pf-sandbox-usw2 || terraform workspace new pf-sandbox-usw2

# Review plan
terraform plan -var-file=workspaces/pf-sandbox-usw2.tfvars.json

# Apply
terraform apply -var-file=workspaces/pf-sandbox-usw2.tfvars.json
```

### Verify Deployment

```bash
# Check pods
kubectl get pods -n atlantis

# Check ingress
kubectl get ingress -n atlantis

# Test health endpoint
curl https://atlantis-pf-sandbox.infra-dev.arcteryx.io/healthz
```

### Components

1. **Atlantis Server**: Runs Terraform commands, posts PR comments
2. **Persistent Volume**: Stores Terraform state and cloned repositories
3. **IRSA Role**: Provides AWS permissions without static credentials
4. **ALB Ingress**: Exposes Atlantis for GitHub webhooks
5. **ConfigMaps**: Store configuration files (repos.yaml, gen-atlantis.py)

## Configuration

### Workspace Configuration

Each workspace is configured via a `.tfvars.json` file:

```json
{
  "region": "us-west-2",
  "env": "pf-sandbox",
  "workspace": "pf-sandbox-usw2",
  "cluster_name": "pf-sandbox-platform-dev",
  "backend_bucket": "arcteryx-pf-sandbox",
  "atlantis_version": "v0.31.0",
  "terraform_version": "v1.11.3",
  "use_github_app": true,
  "github_app_id": "2248528",
  "github_app_installation_id": "93456700",
  "repo_allowlist": "github.com/tienj-arc/*",
  "enable_ingress": true,
  "policy_approvers": ["tienj-arc"]
}
```

### Repository Configuration

Atlantis uses two configuration files:

#### 1. Server-Side Config (`repos.yaml`)

Defines global settings for all repositories:

- Workflow definitions (plan, apply steps)
- Apply requirements (approval, mergeable)
- Policy checks
- Pre-workflow hooks (including git credentials for private modules)

Location: `atlantis/terraform/files/repos.yaml`

**Private Module Access**: The repos.yaml includes a pre-workflow hook that configures git credentials using GitHub App authentication. The hook automatically generates a JWT from the GitHub App credentials, exchanges it for an installation access token, and configures git to use this token for private module access. This is more secure than using a static personal access token.

#### 2. Dynamic Config (`atlantis.yaml`)

Auto-generated by `gen-atlantis.py` script:

- Discovers all Terraform projects
- Creates project definitions
- Configures autoplan triggers

Generated at: Repository root (not committed)

### Filtering Projects

Control which projects Atlantis manages:

#### Enable Specific Workspaces

Edit `atlantis/conf/enabled_contexts.json`:

```json
["pf-sandbox", "omni-dev"]
```

Only workspaces containing these strings will be processed.

#### Skip Components

Edit `atlantis/conf/skip_components.json`:

```json
["test", "experimental"]
```

These components will be completely skipped.

## Usage

### Making Infrastructure Changes

See [WORKFLOW_GUIDE.md](WORKFLOW_GUIDE.md) for detailed workflows.

Quick example:

```bash
# 1. Create branch
git checkout -b feature/add-vpc

# 2. Make changes
vim vpc/terraform/main.tf

# 3. Push and open PR
git push origin feature/add-vpc

# 4. Atlantis auto-plans
# Review plan in PR comments

# 5. Apply changes
# Comment on PR: atlantis apply

# 6. Merge PR
```

### Atlantis Commands

Post these as PR comments:

```bash
atlantis plan                    # Plan all projects
atlantis plan -p vpc-pf-sandbox  # Plan specific project
atlantis apply                   # Apply all projects
atlantis apply -p vpc-pf-sandbox # Apply specific project
atlantis unlock                  # Unlock projects
```

## Custom Docker Image

Atlantis uses a custom Docker image with Python3 for running the config generator script.

### Building the Image

```bash
cd atlantis/docker

# Build
docker build -t atlantis-custom:latest .

# Tag for ECR
docker tag atlantis-custom:latest \
  529088297181.dkr.ecr.us-west-2.amazonaws.com/atlantis:latest

# Push to ECR
aws ecr get-login-password --region us-west-2 | \
  docker login --username AWS --password-stdin \
  529088297181.dkr.ecr.us-west-2.amazonaws.com

docker push 529088297181.dkr.ecr.us-west-2.amazonaws.com/atlantis:latest
```

### Image Contents

- Base: `ghcr.io/runatlantis/atlantis:v0.31.0`
- Added: Python3, PyYAML
- Purpose: Run `gen-atlantis.py` script

## GitHub App Setup

### Creating the GitHub App

1. Go to GitHub Settings → Developer settings → GitHub Apps
2. Click "New GitHub App"
3. Configure:
   - **Name**: `Atlantis-PF-Sandbox`
   - **Homepage URL**: `https://atlantis-pf-sandbox.infra-dev.arcteryx.io`
   - **Webhook URL**: `https://atlantis-gateway.infra-dev.arcteryx.io/events`
   - **Webhook secret**: Generate and save to AWS Secrets Manager

4. Set permissions:
   - Repository contents: Read & Write
   - Pull requests: Read & Write
   - Issues: Read & Write
   - Commit statuses: Read & Write

5. Subscribe to events:
   - Pull request
   - Pull request review
   - Pull request review comment
   - Push
   - Issue comment

6. Generate private key and save to AWS Secrets Manager

### Installing the GitHub App

1. Go to app settings → Install App
2. Select organization
3. Choose repositories (or all repositories)
4. Note the installation ID

### Storing Credentials

```bash
# Store private key
aws secretsmanager create-secret \
  --name arcteryx/pf/atlantis/github-app-private-key \
  --secret-string file://private-key.pem

# Store webhook secret
aws secretsmanager create-secret \
  --name arcteryx/pf/atlantis/github-webhook-secret \
  --secret-string "your-webhook-secret"
```

## IAM Permissions

Atlantis uses IRSA (IAM Roles for Service Accounts) for AWS access.

### Required Permissions

The IAM policy (`iam-policy.json`) grants:

- **S3**: Terraform state bucket access
- **DynamoDB**: State locking
- **EKS**: Cluster management
- **EC2**: VPC, security groups, instances
- **IAM**: Role and policy management
- **Secrets Manager**: Read secrets
- **And more**: Based on your infrastructure needs

### Customizing Permissions

Edit `atlantis/terraform/iam-policy.json` to add/remove permissions:

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": ["s3:GetObject", "s3:PutObject"],
      "Resource": "arn:aws:s3:::${backend_bucket}/*"
    }
  ]
}
```


### State Lock Issues

```bash
# List locks
aws dynamodb scan --table-name terraform-state-lock

# Remove stale lock (use with caution)
aws dynamodb delete-item \
  --table-name terraform-state-lock \
  --key '{"LockID": {"S": "LOCK_ID_HERE"}}'
```

## Maintenance

### Updating Atlantis Version

1. Update version in workspace config:

   ```json
   {
     "atlantis_version": "v0.32.0",
     "atlantis_chart_version": "5.21.0"
   }
   ```

2. Apply changes:
   ```bash
   terraform apply -var-file=workspaces/pf-sandbox-usw2.tfvars.json
   ```

### Updating Terraform Version

1. Update in workspace config:

   ```json
   {
     "terraform_version": "v1.12.0"
   }
   ```

2. Apply changes (Atlantis will use new version for future plans)

### Rotating GitHub App Credentials

1. Generate new private key in GitHub App settings
2. Update secret in AWS Secrets Manager:
   ```bash
   aws secretsmanager update-secret \
     --secret-id arcteryx/pf/atlantis/github-app-private-key \
     --secret-string file://new-private-key.pem
   ```
3. Restart Atlantis:
   ```bash
   kubectl rollout restart statefulset/atlantis -n atlantis
   ```

### Backup and Recovery

Atlantis stores data in a persistent volume:

```bash
# Backup PVC
kubectl get pvc -n atlantis

# Create snapshot (AWS EBS)
aws ec2 create-snapshot \
  --volume-id vol-xxxxx \
  --description "Atlantis backup $(date +%Y%m%d)"

# Restore from snapshot
# 1. Create new volume from snapshot
# 2. Update PVC to use new volume
# 3. Restart Atlantis
```

## Security Best Practices

1. **Use GitHub App** (not Personal Access Token)
2. **Enable HTTPS** with valid certificate
3. **Restrict repo allowlist** to specific organizations
4. **Require PR approval** before apply
5. **Use IRSA** instead of static AWS credentials
6. **Rotate secrets** regularly
7. **Monitor webhook deliveries** for suspicious activity
8. **Use internal ALB** if possible
9. **Enable audit logging** in CloudTrail
10. **Review IAM permissions** regularly
